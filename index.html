Model 3 of 3

Adaptive Testing
via Bloom's Taxonomy

A question difficulty state machine based on Bloom's Taxonomy Levels (BTL). Two consecutive correct answers → level up. One wrong answer → immediate drop. The quiz tracks where you truly are.

// Adaptive BTL State Machine
if (correct && streak ≥ 2) BTL = min(4, BTL+1) // Level UP
if (!correct) BTL = max(1, BTL-1)          // Level DOWN
// Incremental Running Average (O(1) space)
newAvg = (oldAvg × oldCount + newScore) / newCount
No historical scores stored — mathematically exact cumulative average in constant memory.

Why It Works

The Reasoning Behind the Design

🧠
GCN over simple regression

The curriculum is a graph. Topic mastery depends on prerequisite mastery. A GCN captures this dependency via neighbourhood aggregation — a flat NN would ignore the prerequisite chain entirely.

🎲
MC Dropout over deterministic NN

Uncertainty quantification is critical. The system avoids flagging students when its own model is unsure — preventing false intervention alarms without expensive Bayesian retraining.

📚
FAISS + local LLM for RAG

Zero API cost for remedial plans. No student data leaves the machine. Textbook-grounded answers prevent hallucination. Privacy-first architecture by default.

⚡
Gemini for batch generation

Local LLaMA 3 is too slow for 40-question batch generation or full curriculum JSON. Gemini 2.5 Flash handles it in ~2 seconds with perfect JSON structure adherence.

🕸️
Neo4j for curriculum

Cypher queries naturally express prerequisite chains. Adding a new prerequisite is a single MERGE (a)-[:PRECEDES]->(b) — no schema change, no migration needed.

📊
Bloom's Taxonomy as difficulty axis

Universally accepted pedagogical framework. Faculty already design syllabi around it. Maps cleanly to MCQ / Descriptive / Coding question types at BTL 1–4.
